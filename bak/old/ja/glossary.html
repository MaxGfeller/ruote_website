OpenWFEru 用語集

OpenWFE(ru) にある用語を定義・説明するささやかな試み…。

エンジンのコンポーネントは…。

  • Engine (ワークフローエンジン) はRuby のインスタンスです。それにはあなたがワークフローを使うのに必要なメソッドが全て含まれています。まあ、あなたはこれ以外のことにタッチしたくないでしょ? (いくつかの participant を除いて).
  • Expression Map は ‘sequence’, ‘concurrence’ や ‘participant’ などの expression 名を、SequenceExpression, ConcurrenceExpression や ParticipantExpressionなどの実際の実装クラスにマッピングする小さなサービスです。
  • Expression Pool はワークフローエンジン心臓部です。
  • Expression Storage は expression pool の永続的なジョブをします。. 現在、2つの主な実装ができています : “in memory” と “yaml files” です。
  • Participant Map はビジネスプロセス・ワークフローの participant のディレクトリです。このディレクトリのキーとして正規表現も使うことができます。participant map は participant の expression が実際の participant を実行するため(ワークアイテムとそれをディスパッチするため)に参照します。
  • Scheduler は 、’sleep’, ‘cron’, ‘when’ など時間に関する expression を、後のある時間またはさまざまな周期で(cron)実行するために使います。scheduler は OpenWFEru ビジネスプロセスエンジンの外側で" ‘単独で’ 使う":scheduler.html こともできます。
